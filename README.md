## Flagship project

### [Accessibility Audit Automation — Case Study](https://github.com/NguyenThuan-data/accessibility-audit-case-study)

Manual WCAG specialist audits did not scale for release cadence — we built a **confirmation-first MCP workflow** so QA and developers run compliance checks from natural language, without scripting.

- **Team lead (4 people)** for Datacom graduate project — MCP server + Playwright + Axe-core
- **~30s automated scan** vs hours of manual specialist review per release cycle
- **3 report formats** for developers, QA, and stakeholders — code stays private; case study documents process and learning

`accessibility` · `wcag` · `playwright` · `axe-core` · `model-context-protocol`

## Pinned projects

<table>
<tr>
<td width="50%" valign="top">

### [VI_STT_DIAR_PUNC](https://github.com/NguyenThuan-data/VI_STT_DIAR_PUNC)
**What I did:** Vietnam internship — built ASR → diarization → punctuation → LLM summary stack with FastAPI, custom UI, and Docker.

**Impact:** Demo pipeline turns ~20-min consult audio into structured, speaker-labeled transcript + summary.

**Challenge:** Multi-model orchestration on CPU; model weights too large for Git.

**Lesson:** Production ML is integration and honest limits — not notebook accuracy alone.

`speech-recognition` · `fastapi` · `docker`

### [Bank Outcome Prediction](https://github.com/NguyenThuan-data/Bank_Outcome_Prediction_System)
**What I did:** COMP615 coursework → Streamlit app + `batch_score.py` for CSV batch inference.

**Impact:** 89% KNN accuracy on imbalanced dataset (11.7% base subscription rate).

**Challenge:** Making probability and confidence readable to non-technical reviewers.

**Lesson:** Ship something people can click — not just a notebook.

`machine-learning` · `streamlit` · `scikit-learn`

### [SQL Data Warehouse](https://github.com/NguyenThuan-data/SQL-data-warehouse-project)
**What I did:** Bronze/Silver/Gold PostgreSQL warehouse with star-schema gold layer and stored-procedure loads.

**Impact:** Unified CRM + ERP metrics into executive-ready SQL (regional revenue, SaaS leaderboard).

**Challenge:** Messy CSV grains and disciplining data across medallion layers.

**Lesson:** Star schema forces the right business question before writing SQL.

`postgresql` · `data-engineering` · `sql`

</td>
<td width="50%" valign="top">

### [Dynamic Pricing Strategy](https://github.com/NguyenThuan-data/Dynamic_Pricing_Strategy)
**What I did:** Notebook analysis → isolated `pricing_engine.py` → FastAPI + interactive dashboard.

**Impact:** Real-time surge multipliers with 0.8x price floor; [static plots on GH Pages](https://nguyenthuan-data.github.io/Dynamic_Pricing_Strategy/).

**Challenge:** Separating testable business logic from the HTTP layer; two demo surfaces.

**Lesson:** A clean engine layer unlocks full-stack and API work.

`fastapi` · `python` · `data-science`

### [Walmart Sales Analysis](https://github.com/NguyenThuan-data/Walmart_Sales_Analysis)
**What I did:** Notebook → Streamlit BI dashboard with filters + 7-day sales forecaster.

**Impact:** Peak-hour staffing insights (3–8 PM) and category profit concentration for managers.

**Challenge:** Moving from "what happened?" to "what next?" for non-technical reviewers.

**Lesson:** A dashboard explains value faster than raw code.

`streamlit` · `analytics` · `forecasting`
